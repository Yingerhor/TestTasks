ТЗ:
Написать скрипт на bash для мониторинга процесса test в среде 
linux. Скрипт должен отвечать следующим требованиям: 
1. Запускаться при запуске системы (предпочтительно написать юнит 
systemd в дополнение к скрипту) 
2. Отрабатывать каждую минуту 
3. Если процесс запущен, то стучаться(по https) на 
https://test.com/monitoring/test/api 
4. Если процесс был перезапущен, писать в лог /var/log/monitoring.log 
(если процесс не запущен, то ничего не делать) 
5. Если сервер мониторинга не доступен, так же писать в лог.

В соответствии с заданием реализован bash-скрипт, приводимый в исполнение в автоматическом режиме при помощи systemd юнита. Данный юнит запускается со стартом системы ежеминутно при помощи таймера.

Для работы данного скрипта необходимо под пользователем root скопировать или создать 4 файла в следующих директориях:
1. bash-скрипт: /usr/local/bin/test.sh
2. флаг детекции перезапуска: /usr/local/bin/last_state_timestamp
3. юнит: /etc/systemd/system/monitor_test.service
4. таймер: /etc/systemd/system/monitor_test.timer

Далее сделать скрипт исполняемым:
chmod +x /usr/local/bin/test.sh

и активировать юнит и таймер:
systemctl daemon-reload && \
systemctl enable monitor_test.timer && \
systemctl enable monitor_test.service && \
systemctl start monitor_test.timer && \
systemctl start monitor_test.service

После чего начнётся ведение лога в директории /var/log/monitoring.log
